# Food Order App - End-to-End System Simulation

[cite_start]This project is a comprehensive simulation of a food ordering system, built using **Single Page Application (SPA)** architecture[cite: 14, 36].
[cite_start]Developed as part of the Fullstack development track at Lev Academic Center[cite: 1, 3, 5].

## Core Features
* [cite_start]**User Management:** Includes dedicated screens for user registration and login, featuring form validation and credential verification[cite: 13, 33].
* [cite_start]**Menu Operations:** Supports full CRUD functionality (Create, Read, Update, Delete) for menu items, including search and detailed item views[cite: 33, 44].
* [cite_start]**Client-Server Architecture:** Implements a strict separation between client-side logic and server-side simulation using Object-Oriented Programming (OOP)[cite: 15, 77].
* [cite_start]**Asynchronous Communication:** Features a custom **FAJAX** (Fake AJAX) mechanism to simulate real-world network requests using a specialized `FXMLHttpRequest` class[cite: 24, 25, 45, 50].
* [cite_start]**REST API Integration:** Utilizes standard HTTP methods including GET, POST, PUT, and DELETE for data manipulation[cite: 39, 44].
* [cite_start]**Data Management:** All data is persisted in **Local Storage** and accessed exclusively through a defined DB-API[cite: 22, 23, 63].

## Technical Stack
* [cite_start]**Languages:** HTML5, CSS3, and JavaScript (ES6+)[cite: 3, 28, 71].
* [cite_start]**Data Format:** JSON for structured data exchange between client and server[cite: 24, 52].
* [cite_start]**Architecture:** Component-based design with professional folder separation (Client, Server, DB, Network)[cite: 76, 77].
* [cite_start]**Version Control:** Managed and deployed using Git.

## Execution
To run the application, open `index.html` in any modern web browser. [cite_start]The system will initialize all components and simulate a live network environment[cite: 51, 76].
